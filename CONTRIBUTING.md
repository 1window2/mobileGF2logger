# Contributing to mobileGF2logger

Keep each commit limited to one logical responsibility. Do not commit packet captures, exported player data, credentials, keys, generated logs, APKs, or build caches.

## Commit tags

- `feat:` new functionality
- `fix:` bug fixes
- `docs:` documentation-only changes
- `chore:` build, dependency, or repository maintenance
- `refactor:` internal changes without new behavior
- `style:` formatting-only changes
- `test:` test additions or corrections

## Format

```text
<type>: <concise imperative description>
```
